The Box Status Module

The purpose of the box status module is to allow an EPICS IOC to check is any network capable box is connected/online, essentially by pinging the box. Any box that can respond to a ping is suitable, e.g. EPICS IOC host boxes, MOXA boxes, network switchs, motion controllers.

The Box Status module has been developed for and on Linux. It could be extended for other operating systems.

Including the Box Status module into an EPICS IOC

Within the EPICS IOC's configure/RELEASE file: 
BOXSTATUS=$(BUNDLE)/epics/boxStatus

Within the EPICS IOC's make file (<top>/IOC_Name_App/src/Makefile): 

IOC_Name_DBD += devBoxStatus.dbd
IOC_Name_LIBS += devBoxStatus

And within the IOC's st.cmd file (after iocInit): 

# when built directly into the IOC example (easy build)
dbLoadRecords ("db/Box_Status.template", "BOX=SR16MB03DEV01, FUNCTION=Primary Widget, LOCATION=Sever Room 2")
dbLoadRecords ("db/Box_Status.template", "BOX=SR16MB03DEV02, FUNCTION=Secondary Widget, LOCATION=Tunnel Roof")
or 
# when not built directly into the IOC examole (bundle build)
dbLoadRecords ("$(BOXSTATUS)/Box_Status.template", "BOX=SR16MB03DEV01, FUNCTION=Primary Widget, LOCATION=Sever Room 2")
dbLoadRecords ("$(BOXSTATUS)/Box_Status.template", "BOX=SR16MB03DEV02, FUNCTION=Secondary Widget, LOCATION=Tunnel Roof")
or probably better to use a substitution file if more than a limited number of devices ned to be specified.
